# PRD｜工作室静态官网（简单大气上档次）

- 任务ID：WEB-20260317-001
- 需求方：<at user_id="ou_352a0ad3b8ecf37561ca52b8587898f8"></at>
- 类型：新建（静态站）
- 优先级：P1
- 参考风格：腾讯官网（https://www.tencent.com/zh-cn/）
- 部署方式：GitHub Pages（Q2=1）

## 1. 背景与目标
为工作室搭建一个“简单大气上档次”的静态官网，用于品牌展示、对外背书、呈现案例与服务能力，并提供明确的联系入口。

## 2. 目标用户
- 潜在客户（希望快速了解工作室能力、案例、联系方式）
- 合作方/渠道（评估可信度）
- 求职者（若后续扩展栏目）

## 3. 信息架构（Q1=B：标准 5 页）
### 3.1 页面列表
1) 首页 Home
2) 案例 Case/Works
3) 服务 Services
4) 关于 About
5) 联系 Contact

### 3.2 全站公共区域
- 顶部导航：Home / Works / Services / About / Contact
- 右上角 CTA：联系/咨询（跳转 Contact 或弹出联系方式区域）
- 页脚：版权、备案位（可选）、社媒/邮箱（可选）

## 4. 关键页面内容与交互
### 4.1 首页
- 首屏：品牌名/一句话 Slogan + 1 张高质感大图或抽象背景（可用渐变/噪点纹理）
- 能力概览：3-6 个服务亮点卡片
- 精选案例：展示 6-9 个案例缩略图（点击进入案例详情或跳转到案例列表页）
- 信任背书区：合作客户/数据指标/奖项（若无素材先预留占位）
- 底部 CTA：立即联系（按钮 + 联系方式入口）

### 4.2 案例（列表页）
- 案例卡片：封面图 + 标题 + 标签（如品牌/活动/设计/开发）
- 支持筛选：按标签筛选（可选，建议首版做“标签展示+前端筛选”即可，无需复杂搜索）
- 案例数量不足时：允许先放“Coming soon”占位卡

### 4.3 服务
- 服务条目：每项包含「标题 + 简述 + 交付物示例」
- 服务流程：3-5 步（如需求沟通→方案→制作→交付→迭代）

### 4.4 关于
- 工作室介绍：理念/优势/团队介绍（可选）
- 资质/过往经历（可选，先预留模块）

### 4.5 联系
- 联系方式：微信二维码/邮箱/电话（至少一种明确可用）
- 留资入口：表单（静态站首版建议：mailto 链接或跳转第三方表单；若必须站内表单，需补充后端或表单服务）
- 地址信息（可选）

## 5. 视觉与体验规范（对齐“腾讯系”高级感）
- 设计关键词：留白、栅格、秩序、克制动效、低饱和（黑白灰为主，少量品牌色点缀）
- 字体：系统字体优先（避免首版引入复杂字体加载）；如需更高级可用思源黑体/Inter（需评估性能）
- 动效：轻量（hover、淡入、滚动进入），避免花哨
- 响应式：PC 优先 + 移动端适配
- 性能：首屏快速渲染，图片懒加载，Lighthouse 基础分达标（见验收）

## 6. 技术与交付约束
- 静态托管：GitHub Pages
- 不包含：登录系统、CMS 后台、复杂表单后端、支付/下单
- 资源要求：案例图需由需求方提供（或先占位）

## 7. 验收标准（可测试）
### 7.1 功能验收
- 全站 5 个页面可访问，导航互通，无死链
- Works 支持展示至少 6 个案例卡片（不足可占位）
- Contact 页至少提供 1 个可用联系渠道（微信二维码/邮箱/电话其一）

### 7.2 适配与兼容
- Chrome / Edge 最新版：布局与交互正常
- iPhone 常见尺寸（390×844）与 Android 常见尺寸（360×800）下无明显错位、文字溢出

### 7.3 性能与质量
- Lighthouse（移动端）目标：Performance ≥ 80，Accessibility ≥ 90，Best Practices ≥ 90，SEO ≥ 90（首次版本可接受个别项小幅偏差，但需解释原因）
- 图片体积控制：单张建议 ≤ 500KB（可按实际素材调整）

### 7.4 部署验收
- 提供 GitHub 仓库地址
- GitHub Pages 部署成功：给出线上访问 URL
- README 包含：本地启动方式、构建/发布方式、内容替换说明（案例/文案/联系信息在哪里改）

## 8. 待补充素材清单（需求方提供）

- 联系方式：至少 1 个（推荐：微信二维码 + 邮箱）
- Logo（可选；没有则先用文字 Logo）
